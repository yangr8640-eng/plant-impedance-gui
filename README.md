BMI 硬件 - Windows 版四路电阻曲线 GUI

使用方法：
1. 先安装 Python 3，并在安装时勾选 Add python.exe to PATH。
2. 双击“启动电阻曲线GUI_Windows.bat”。
3. 启动器会自动检查并安装 pyserial 串口依赖。
4. 通过拓展坞连接最多四个相同硬件板子，打开后点击任一通道里的“刷新串口”。
5. 顶部页签对应四个独立通道，格式为“通道1 · COM3 · 绿萝01”。切换页签只切换当前查看的植物，其他通道会继续在后台读取、绘图和记录。
6. 在每个通道中选择对应的 COM3、COM4 等串口，并选择和 SSCOM 一致的波特率。
7. 开始测量前，每个通道单独填写“epoch 数量”和“每个 epoch 时长”（可选秒、分钟或小时）。GUI 会显示计算后的总时长。
8. 填写“记录间隔(s)”“横轴范围(h)”和植物编号、温度、湿度、电极位置、土壤状态。
9. 如果曲线被自动放得太大，可在当前通道的“Y格差(Ω)”中填写每个 y 轴单元格的电阻差，例如 10 表示每格 10Ω；留空表示自动。
10. 点击当前通道的“连接并开始测量”。epoch 设置会在测量期间锁定；需要四路同时测量时，依次切到四个页签并分别连接不同串口。
11. 可使用当前通道的“放大”“缩小”“重置缩放”查看曲线细节；在图像上拖移可平移横轴查看更早时间段，鼠标滚轮可缩放；鼠标移到曲线点附近会显示该点的时间和电阻。
12. 实验结束后，当前页签的“导出记录CSV”只导出该通道的原始记录，“导出PDF”只导出该通道的实验报告。

时间轴说明：
- 每个通道的 x 轴会从 0 秒显示到“epoch 数量 × 每个 epoch 时长”得到的总时长。
- 计时从点击“连接并开始测量”且串口连接成功后开始，不再等待第一条数据。
- GUI 默认每 60 秒记录 1 个有效电阻点；这个间隔可在每个通道里单独修改。
- GUI 默认显示最近 6 小时曲线；可通过“横轴范围(h)”调节横轴显示窗口，并用拖移查看历史时间段。
- 每个 epoch 到时后，GUI 会立即关闭并保存该 epoch 的原始 CSV，然后在不断开串口的情况下继续下一个 epoch。
- 自动文件名以 `_epoch1.csv`、`_epoch2.csv` 等结尾。即使某个 epoch 没有收到有效数据，也会保存一个带表头的 CSV。
- 最后一个 epoch 结束后，该通道会自动停止读取串口或停止模拟数据，其他通道不受影响。
- GUI 中的图像 y 轴、顶部统计值和悬浮提示统一使用 kΩ。
- 每个被记录的点会立即写入当前 epoch 的 CSV，其中同时保存 epoch 编号、epoch 内序号、epoch 内耗时和整体实验耗时。

注意：
- 同一个 COM 口同一时间只能连接到一个通道；如果误选重复串口，GUI 会提示该串口已被其他通道连接。
- 如果 SSCOM 正在连接某个串口，请先在 SSCOM 中关闭该串口，再回到 GUI 对应通道里点击“连接”。
- 拓展坞建议使用供电稳定、数据口充足的型号；如果四个板子中有某一路不出现 COM 口，优先检查 USB 线、供电和驱动。
- 如果 pyserial 自动安装失败，可以手动运行：
  python -m pip install pyserial
- 长时间测试时，曲线点数主要取决于“记录间隔(s)”；短时间测试想看更流畅曲线时，可让硬件每 10-50 ms 发送一次并调小记录间隔，例如：
  R=1023.5 ohm
- “导出记录CSV”会把当前通道的所有 epoch 文件合并成一个 CSV；各 epoch 的自动保存原文件不受影响。
- 24 小时测试建议：例如设置 4 个 epoch、每个 6 小时，记录间隔填 60 或更长，横轴范围先填 6；需要看全程时改成 24，并确认电脑不会自动休眠。
- PDF 不再导出每个时间点的原始数据，避免报告过长。
- 如果芯片一行里同时输出 i、Rs、Rr，例如 “i=608 Rs=142735 Rr=369”，GUI 会优先把 Rs 当作电阻值绘图。
