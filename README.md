# 实时电阻曲线

这是一个用于电阻测量硬件的本地 Web 应用。它通过浏览器的 Web Serial API 直接读取串口数据，并以时间为 x 轴、电阻为 y 轴实时绘制曲线。

## GitHub 下载分支

这个仓库会同时维护三个分支：

- `main`：完整项目说明，包含 Windows、macOS 和网页版本入口。
- `windows`：只保留 Windows GUI 需要的文件，适合 Windows 用户直接下载 ZIP。
- `mac`：只保留 macOS GUI 需要的文件，适合 macOS 用户直接下载 ZIP。

在 GitHub 页面左上角切换到对应分支后，点击 `Code` -> `Download ZIP` 即可下载。

## 为什么不是直接连接 SSCOM

SSCOM/SerialInsight 主要是串口调试工具，本身负责打开串口、显示接收内容和发送数据。大多数系统里，同一个串口通常只能被一个程序独占打开。因此这个应用推荐直接连接你的硬件串口，而不是再去连接桌面上的 SSCOM。

建议工作方式：

1. 先用 SSCOM 验证硬件能正常输出电阻值。
2. 在 SSCOM 中关闭串口连接。
3. 打开本应用，选择同样的串口和波特率。
4. 实时查看电阻曲线。

## Windows GUI 运行方式

如果硬件串口连接在 Windows 电脑上，使用 Windows 版 GUI。

准备工作：

1. 安装 Python 3。
2. 安装时勾选 `Add python.exe to PATH`。
3. 把整个项目文件夹复制到 Windows 电脑。

然后双击：

```text
启动电阻曲线GUI_Windows.bat
```

这个启动器会自动检查并安装串口依赖 `pyserial`，然后打开 GUI。

打开后：

1. 点击“刷新串口”。
2. 选择硬件对应的 `COM3`、`COM4` 等串口。
3. 选择和 SSCOM 一致的波特率。
4. 在实验变量区的“测试时长(h)”中填写本次实验时长，例如 `24` 表示 24 小时，`0.5` 表示 30 分钟。
5. 在“记录间隔(s)”中填写实际保存和绘图的间隔。植物阻抗变化较慢时建议 `60`、`300` 或 `3600`；填 `60` 表示每分钟保存 1 个点。
6. 在“横轴范围(h)”中填写曲线当前显示的时间窗口。默认 `6` 表示显示最近 6 小时；填 `24` 可看完整 24 小时；填 `1` 可放大看最近 1 小时。
7. 填写其他实验变量：实验时间、温度、湿度、电极位置、土壤状态和植物编号。
8. 如果曲线被自动放得太大，可在“Y格差(Ω)”中填写每个 y 轴单元格的电阻差，例如 `10` 表示每格 `10Ω`；留空表示自动。
9. 可使用“放大”“缩小”“重置缩放”查看曲线细节；在图像上拖移可平移横轴查看更早时间段，触控板滚动或鼠标滚轮可缩放；鼠标移到曲线点附近会显示该点的时间和电阻。
10. 点击“连接”。
11. 查看实时电阻曲线和接收日志。
12. 点击“导出记录CSV”，保存已记录的数据；也可点击“导出PDF”，保存包含实验变量、平均值、极值和曲线图的实验报告。

计时从第一条有效电阻数据开始。到达设定测试时长后，GUI 会自动停止读取串口或停止模拟数据，曲线不会继续追加。
GUI 中的图像 y 轴、顶部统计值和悬浮提示统一使用 `kΩ`。
桌面 GUI 默认每 60 秒记录 1 个有效电阻点，并会把每个记录点立即写入 `output/植物阻抗自动记录_*.csv`。
桌面 GUI 默认显示最近 6 小时曲线；可通过“横轴范围(h)”调节横轴显示窗口，并用拖移查看历史时间段。

如果 SSCOM 正在连接同一个串口，请先在 SSCOM 中关闭串口，否则 Windows GUI 也可能无法打开。

也可以手动安装依赖并启动：

```bat
python -m pip install pyserial
python ResistanceGUI_windows.py
```

## 网页运行方式

macOS 上可以直接双击：

```text
启动电阻曲线网站.command
```

它会启动本地服务并自动打开浏览器。请保持弹出的终端窗口打开，关闭窗口后网站会停止。

也可以手动在项目目录中启动本地服务：

```bash
python3 -m http.server 5173
```

然后用 Chrome 或 Edge 打开：

```text
http://localhost:5173
```

Web Serial API 需要浏览器支持，并且页面需要在安全上下文中运行。`localhost` 可以满足这个要求。

## 硬件输出格式

每次测量建议输出一行文本，以换行符结尾：

```text
R=1023.5 ohm
R=1.024 kΩ
resistance: 998.2 Ω
Rs=142735, Rr=369, i=608
```

应用会优先提取 `R=...`、`Rs=...`、`resistance: ...`、`电阻=...` 这类字段；如果没有这些字段，再提取每行中的第一个数值。这样一行里同时包含 `i=...`、`Rs=...`、`Rr=...` 时，会把 `Rs` 作为电阻值。可识别这些单位：

- `Ω` / `ohm` / `欧` / `欧姆`
- `kΩ` / `kohm` / `千欧`
- `MΩ` / `Mohm` / `兆欧`
- `mΩ` / `mohm` / `毫欧`

如果没有单位，默认按欧姆处理。

## 曲线和长时间记录

GUI 会尽量快速读取串口数据，但桌面 GUI 只会按“记录间隔(s)”保存和绘图。植物 24 小时慢变化测试建议把记录间隔设为 60 秒或更长，并用“横轴范围(h)”在全程和局部细节之间切换；如果想看短时间快速变化，再把记录间隔调小，并在硬件/下位机端提高发送频率。

推荐：

- 每 10-50 ms 发送一次电阻值，约 20-100 Hz。
- 每条数据以 `\n` 或 `\r\n` 结尾，例如 `R=1023.5 ohm\n`。
- 波特率可以使用 `115200` 或更高，前提是硬件和 GUI 设置一致。
- 如果发送频率很高，接收日志会快速滚动；曲线仍会正常显示。

## 功能

- 串口参数选择：波特率、数据位、停止位、校验位
- 实时曲线：电阻随时间变化
- 自动或固定 y 轴范围
- GUI 显示单位固定为 kΩ，内部仍按 Ω 保存和计算
- 简单平滑滤波
- 当前值、平均值、最小值、最大值
- 模拟数据模式，方便无硬件测试界面
- 记录 CSV 导出：复制完整自动记录文件，并附带接收时间、解析出的电阻值和原始文本行
- PDF 导出：实验变量、平均值、极值和曲线图，不导出逐点原始数据

## 无法连接串口时

先看页面左侧串口区域里的诊断：

- 浏览器串口应显示“支持”。如果不是，请换 Chrome 或 Edge。
- 安全上下文应显示“正常”。如果不是，请用 `http://localhost:5173` 打开，不要直接双击 HTML。
- 页面来源建议是 `http://localhost:5173`。
- 最近错误会提示具体原因。

常见原因：

- SSCOM 仍然打开着同一个串口。请先在 SSCOM 里点“关闭串口”。
- 浏览器弹出的串口选择窗口里没有设备。请检查 USB 线、硬件供电、USB 转串口驱动。
- 选择了串口但打开失败。通常是端口被其他软件占用，或设备刚插拔后系统还没有准备好。
- 波特率不一致。连接可以成功，但不会收到正确数据；请和 SSCOM 里能正常显示时的波特率保持一致。

## 已连接但没有曲线时

看左侧诊断里的“收到字节”和“解析成功”：

- 收到字节一直是 0：网页没有收到硬件数据。请确认硬件正在主动发送、串口选对、波特率和 SSCOM 一致。
- 收到字节增加，但解析成功是 0：网页收到了数据，但没有识别出电阻值。看“接收日志”里的原始内容，把其中一两行发给开发者即可调整解析规则。
- 接收日志里显示“未解析”：说明数据格式暂时不匹配。推荐让硬件每条数据带换行，例如 `R=1023.5 ohm\n`。
- 硬件只发送 `\r` 回车、不发送 `\n` 换行也可以；页面已经兼容。
