# WHEELTEC C100/C70 植物定时拍照工具

这是一个优先适配 Windows 的双摄像头拍照程序。GUI 提供两个独立相机通道，每个通道都有 USB 摄像头识别、目标设备绑定、1080p 实时画面、固定间隔拍照、每日固定时间点拍照和运行日志。

WHEELTEC C100/C70 是 UVC USB 摄像头，在 Windows 中属于“相机”设备，不会显示为 `COM3`、`COM4` 这样的串口。因此 GUI 的交互方式与植物阻抗 GUI 类似，但下拉框识别的是 USB 摄像头设备，不是串口。

## GUI 启动

在 Windows 中双击：

```text
start_camera_gui_windows.bat
```

启动器会检查并安装 `opencv-python`、`cv2-enumerate-cameras` 和 `Pillow`。也可以手动运行：

```powershell
python -m pip install -r requirements.txt
python camera_gui.py
```

## 使用步骤

1. 将两个 WHEELTEC 摄像头连接到 Windows 电脑，后续尽量保持 USB 接口不变。
2. 点击“刷新全部设备”，分别进入“相机 1”和“相机 2”标签页。
3. 在每个标签页选择一个摄像头，点击“连接并预览”，通过实时画面确认对应关系。
4. 为两个通道填写不同的“照片标识”，例如 `左侧植株` 和 `右侧植株`。
5. 分别设置固定间隔、固定时间点或同时启用两者。
6. 设置保存目录后分别启动计划；也可以使用顶部的“连接全部”和“启动全部计划”。GUI 固定使用 PNG 无损保存。

需要选择其他摄像头时，先停止并断开对应通道，然后点击“更换绑定”。标签页切换不会停止后台预览或拍照计划。

## 保存规则

图片先按日期分组，再按相机通道和照片标识分目录保存：

```text
output/2026-07-14/camera1_左侧植株/plant_camera1_左侧植株_20260714_120000_123_interval.png
output/2026-07-14/camera2_右侧植株/plant_camera2_右侧植株_20260714_120000_127_interval.png
```

即使两个相机在同一秒拍照，目录和文件名中的 `camera1` / `camera2` 也能保证照片不会混淆或互相覆盖。

程序会为每个摄像头请求 `MJPG 1920 × 1080 @ 10fps` 视频流。植物定时观察不需要 30fps，将预览帧率降到 10fps 可以显著降低双摄像头同时连接时的 USB 带宽压力，但不会降低保存图片的 1080p 分辨率。GUI 固定编码为 PNG，并将 PNG 压缩级别设为 0，因此不会再进行 JPEG 有损压缩；保存结果相对于 OpenCV 收到的画面帧是无损的。

GUI 每 2 秒在后台重新扫描一次尚未连接的摄像头，扫描不会卡住界面。Windows 上会同时检查 DirectShow 和 Media Foundation，优先使用 DirectShow，打开失败时自动改用 Media Foundation。对于扩展坞刚接入时的短暂漏报，GUI 会复核后才将设备从列表移除。“连接全部”会等待第一个摄像头建立视频流后再连接第二个，避免两个相同设备同时初始化。

扩展坞场景下，建议先接好带独立供电的扩展坞，等 Windows 设备管理器中出现摄像头后再启动 GUI。如果是两路 1080p，仍建议两个摄像头分开接入不同的 USB 3.x 控制器；设备管理器能识别只代表 USB 设备已枚举，不代表扩展坞的视频带宽和供电一定足够。

触发来源保留在文件名中：

- `interval`：固定间隔
- `timepoint`：固定时间点
- `both`：两种触发在同一时刻到期
- `manual`：点击“立即拍照”

同一秒内的计划触发会去重。GUI 会显示成功数、失败数、下一次触发时间和最近保存路径，并把日志写入 `camera_gui.log`。

## 设备安全规则

- 两个通道不能选择同一个物理摄像头。
- 程序优先根据设备路径区分两个具有相同 `VID:PID` 的摄像头，不依赖会变化的 Windows 摄像头序号；当后端没有单独给出 VID/PID 时，会从 Windows 设备路径中回填。
- 目标摄像头断开后，后台会持续寻找同一硬件身份，不会切换到电脑内置摄像头。
- 两个相同型号摄像头首次绑定时会记录各自的设备路径；无法唯一确定时拒绝连接。
- 如果更换 USB 接口导致设备路径变化，请在对应通道点击“更换绑定”并重新确认画面。
- 两个通道的绑定和拍照设置都保存在自动生成的 `camera_gui_config.json` 中。

## 命令行模式

原有无界面定时程序仍保留，但一次进程只控制一个摄像头：

```powershell
python camera_timer.py --list-cameras
python camera_timer.py --camera-id 0c45:6366 --interval-seconds 60 --timepoints 08:00,12:00,18:00
```

示例中的 `0c45:6366` 必须替换为实际摄像头的 `VID:PID`。
